### 如何写好一个简历

> 简历是一块敲门砖，它将全权代表你去突破HR的第一道防线。即使它不是金的，你也应该想办法使它看起来金光闪闪，不然 HR 怎么会注意到你呢？

正常情况下，一份简历大致分为5个模块：

* 个人信息
* 教育经历
* 技能列表
* 工作经历
* 其他

以上顺序能够让HR由浅至深的快速了解求职者的优势和技能，也能够使面试官快速捕捉到他想了解的信息。

#### 个人信息

![个人信息](https://tva1.sinaimg.cn/large/007S8ZIlly1gec8qa3jn7j30je07v0t8.jpg)

个人信息的三要素：姓名、电话、邮箱。这些信息主要是为 HR 以及面试官提供能够联系到你的途径，如果是因为电话或者邮箱联系不到你而失去面试的机会，那就太亏了。如果方便的话你也可以留下自己的微信或者 QQ 。以防出现电话联系不到你的情况。其他的一些信息例如政治面貌、年龄、是否单身等等信息其实就没必要在简历上展示。

如果你是社招，那么可以把下面的教育经历直接省略，并填写毕业院校就行。另外如果你有 GitHub 或者博客的链接也可以加进去，以达到美化的目的。如果你的 GitHub 上面没有几个 commit 也没有几个 start 的话、或者你的博客没有几篇文章，那还是别放上去了。

#### 教育经历

![教育经历](https://tva1.sinaimg.cn/large/007S8ZIlly1gec8sgweh5j30je074t98.jpg)

教育经历就将学校、专业、入学时间/毕业时间写清楚就可以。教育经历其实对于实习生/应届生来说是重点信息，对于社招的同学这块可作为非重点一带而过。

#### 技能列表

![技能列表](https://tva1.sinaimg.cn/large/007S8ZIlly1gec8ws07i0j30jf0813zk.jpg)

HR 通常会采用在简历中搜索关键字的方式来筛选简历。因此技能列表把自己最擅长的工具按分类列上去，熟悉度由于容易受主观因素的影响，见仁见智，所以不用写。

当然只要你写在上面的技术栈都必须是你了解或者掌握的，否则被随便问两个问题就不会了，就变成了减分项目。技能列表请慎重对待，有可能是你的加分项，也有可能是你的减分项。

#### 工作经历

![工作经历](https://tva1.sinaimg.cn/large/007S8ZIlly1gec9dnm52yj30jd0e2gmz.jpg)

工作经历对于开发者来说是最重要一部分了，因为面试官大多会根据你所描述的项目中的某个知识点进行询问。所以这一部分该怎么写？要写什么？要怎么包装？可参考后面有关包装简历部分的描述。

#### 其他

![其他](https://tva1.sinaimg.cn/large/007S8ZIlly1gec9lpc6fuj30j9059aah.jpg)

这里就到了展示你自己的部分了，假设上述个人简介里面你写了你的 GitHub 地址，并且恰好其中某个开源项目还挺厉害，那么这里你就可以对其进行简单描述一下。又或者你的某个社区粉丝数达到了十几万、获得了什么称号也能写在上面。总而言之怎么牛逼怎么来。

### 如何写工作经历？

上面我们说了工作经历是我们简历中最重要的一部分，那么对于这一部分的内容进行包装和编写呢？接下来就教大家如何通过几个特定的方法而使你的经历看起来特别牛逼！

#### 包装简历是对是错？

有一部分可能并不屑于包装自己的简历，认为求职就应该实事求是，做了什么就写什么。或许还有一部分人认为包装简历就是欺骗别人，过不去自己心里的那一关。接下来我们就来讨论一下包装简历是对是错。你只有对“包装”这个词放下戒心，才能够更加用心的“包装”自己的简历。

为什么公司更倾向于将面试机会给那些简历上**看似**做过很多很厉害的事情的人呢？因为他们的简历可能**撒谎**了。为什么不能所有人都把自己**真实**的经历写在简历上呢？对于这个问题，只要想通一点就豁然开朗了——**没有人可以把自己真实的经历写在简历上**

人的经历是立体的，是多维的。假设你做一件事情可能需要耗费三个月的时间，中间遇到了很多困难，但你一一克服了，并付出了很多努力，最后得到一个成果。这就是所谓最真实的经历，但是你并不可以把这个经历都摆到纸面上来。你讲的每一个故事，每一段介绍，都是你这段经历的投影，是它的一部分。更何况，在简历里，你可能只有两三行去描述这一段经历，这时候就需要你摘取这段经历里更加打动人的部分去描述了。这就是所谓的**包装**。

举个例子，A 和 B 用相同的时间完成了同样的事情，并用同样的篇幅去描述这段经历，但由于简历包装描述的不同，A 可能会比 B 看起来更优秀。这就是所谓写简历的技巧。不光简历如此，在平时生活中有的人可以在五分钟之内让你对他产生兴趣，即使他可能不是超级神牛。也有的人虽然有很多好玩的经历，却是茶壶煮饺子—倒不出。

因此在我的角度看来，所谓简历的**包装**，是合情合理的，甚至还是必不可少的。你要知道一个挑选人才的“猎人”，在你的简历上停留的时间最多也就是一分钟，那么你如何利用这一分钟、一页纸把更完整更优秀的你展示给对方，让对方对你产生兴趣，这就是你的主要任务。

而简历的真实性也是应该遵循的原则。切不可背道而驰，要在真实的基础上对简历进行包装美化，世界上有很多的`Half Truth`，你说的东西是真实的，只是讲了一半。在简历里就可以利用这个道理。比如之前在公司做了一个很厉害的项目，但是另外一个组做的比你的更厉害，最后没用你的。你完全可以只描述前半部分，完成了什么、里面的技术关键点是什么，而忽略后面被取代的部分。

这东西和相亲是一样的，你不能自己明明是个穷光蛋但是却说自己是个千万富翁，这迟早会被拆穿，但是你是个保安队长，手下有三个保安，说自己是个小领导，管着三五个人，那还是无伤大雅的。

#### 如何包装自己的简历？

上面我们对于“包装”这个词进行了解释，那么接下来我们就开始讨论如何包装自己的简历，让自己的简历在众多简历中脱颖而出，为自己赢得面试的机会。

##### 回想自己的经历，找到闪光点

> 本来我对自己的了解是模糊的，质疑的，经过这几次多角度多层次的分析，让我更加清晰自己的人格特质和闪光点，增加了我的自信。

程序员的日常工作其实就是增删改查，那么如果你在简历中还是写增删改查那么基本上你与这场面试也就无缘了。所以就需要你在增删改查过程中找到属于自己的闪光点。仔细思考自己的闪光点不仅仅是为了能够增加面试机会，也是为了能够增强自己的自信心。

举个例子，假设还是 A 和 B 两个人做了同样一个后台管理系统供业务人员使用，A 是这么描述的：主要负责后台管理系统的开发，供业务人员使用，为业务人员提供了便利。B 是这么描述的：后台核对几百万的数据，从**5分钟**时间优化到了**30秒**。那么如果你是面试官你会对哪个人感兴趣呢？显而易见肯定是 B。而两个人说的都是事实，那么 A 的错误在哪呢？其实就是不会发掘自己的闪光点，在工作中肯定会或多或少的碰到一些问题，那么在解决这些问题的时候你是怎么想的，怎么做的，你利用了什么技术解决的这个问题，最后给公司、给系统、给组内带来的影响是什么。仔细想想肯定能挑选出属于自己的闪光点。

##### 移花接木

> 假作真时真亦假，真作假时假亦真。

有些人可能看了第一点后自己仔细想了想，最后发现确实没有什么闪光点啊。那么这时候就需要你要移花接木这一招。

每个公司线上的业务或多或少会发生过几次线上的问题，可能是由于代码问题或是因为 JVM 的配置不当导致的，亦或是大数量突然的到来导致的服务不可用。那么碰到问题时肯定会有人去处理解决，最终问题都会被基本解决。就算轮不到你来解决这个问题那么你也要积极的参与进来，弄清楚这个问题为什么发生，事后向同事请教是怎么解决的，里面涉及到的知识点有哪些，这些你都可以记录下来。在简历中写出来就行，因为面试官大多只在乎他问你的知识点你有没有答上来，而很少深究这段经历到底是不是你的。

但是最重要的一点就是你一定要弄清楚这里面的知识点。给大家举个例子比如在线上发生了频繁 GC 而导致服务响应过慢，那么导致频繁 GC 的原因有很多，而面试官看到 GC 基本上都会问 JVM 的模型、内存分配与回收策略等等的问题。所以这一系列的问题你都要搞清楚，最好是做到自己在本机模拟一遍此次问题的发生经过。然后你再试着用同事的解决办法进行解决，最后再思考有没有更好的办法去解决这个问题。如果这些你都想清楚、搞明白了，那么这段经历就是属于你的。

##### 无中生有

> 天下万物生於有，有生於无。

有的人看了上面的两点，可能会说我一没有闪光点，二也不想找同事问，那我该怎么办？那么这时候就可以用到无中生有这一招。

其实无中生有这一招和上面的移花接木有异曲同工之妙，只不过移花接木是要你和同事和身边的人取经，而无中生有则是要你向全世界的人取经。现在网络这么发达，有很多具有分享精神的程序员会在网上分享一些他们在日常开发过程中碰到的问题以及解决方法。那么这时候你就可以通过在网上积累一些实战的经验，但是还是最重要的一条**一定要自己在本机还原一下事故现场**，然后在问题中寻找知识点。

或者是你自己新学习了一个知识点，想对某个业务进行优化，但是迫于种种问题，例如与其他组的协调问题或者组长的不支持，所以导致你是无法进行这个优化的，那么你可以在自己本机搭建一套相关的环境，可以自己试着优化一下，看是否能够给这套业务带来性能的提升，在优化过程中有没有引出其他的问题，这一系列的过程都是你学习的过程，最后虽然你没有真正在业务中优化，那么你也可以将这段经历写入到简历中去。这也是一种无中生有。

##### 有据有理

> 用数据说话，才能让人更信服你。

在写自己的工作经历时，能够用数据化说明的情况尽量用数据来表达。直接举例说明：

* 同事小李描述道：通过我的优化，使对账时间大大减少了。
* 同事小王描述道：通过我的优化，对账时间从之前的**5分钟**优化到了**30秒**。

如果你是面试官，假如这两个人都是相同的经历，那么你会下意识的对哪个人的描述产生兴趣呢？显而易见通过数据能够使你的经历更加具象化，也能够使人更加信服。所以我们在描述我们经历的时候要学会用数据来描述一些场景信息。


### 技术面试

经过了前面的简历关拼杀，终于到了技术面试这个环节了。这个环节就靠你的真才实力了，当然在技术面试的时候也会有些运气的成分，有可能你和这个面试官聊得来，他问的问题正好是你熟知的，那么天时地利人和你都占了，通过当然也是必然的了。但是如果你碰到一个和你气场不和的面试官，无论是提问还是聊天都感觉有一丝的合不来的感觉，那么你这场面试多半是要凉凉了。

所以技术面试七分靠实力，三分靠运气。运气部分我们无法掌握，那么我们就要保证在我们的实力部分不能丢分。而技术面试也分为两部分。

* 准备阶段：在投简历前，我们就开始准备相关的专业知识。
* 交流阶段：和面试官交流时注意一些方法技巧，以及如何将问题引导到我们熟悉的部分。

那么接下来我们就从这两方面入手讨论一下如何应对一场技术面试。

#### 准备阶段

> 不打无准备之仗

##### 随时都准备着

> 最好的准备，就是时刻准备着

随时准备并不是鼓励频繁地跳槽，而是要有随时有跳出舒服圈的准备，也许求职者已经很满意现在的工作以及薪酬，觉得习惯而且安逸。但是如果某天我们由于某些原因而处于被动的位置，我们还能顺利找到这样或者更好的工作吗？我建议各位，每两三个月可以去面试一两家公司，因为你已经有不错的工作了，所以可以带着轻松的心态去面试，同时也可以增加面试的经验。而且你在面试的过程中一方面可以积累属于自己的题库，另一方面也可以查漏补缺。

##### 技术基础

既然我们是技术面试，那么当然应该着重准备技术方面的知识。有关计算机方面的基础知识主要包括：编程语言基础、算法基础、计算机网络、操作系统、数据库五大部分。面试之前的复习阶段是非常重要的，通常大家会通过阅读各类的书籍来补充自己的基础知识，但是计算机方面的书大多都是难且多，而且书的质量也良莠不齐。所以只是无目的的看书来进行准备的话那么就相当于是没有准备。我们可以带着问题去看书或者根据网上提供的面试题去有目的性的看书。下面大概介绍一下技术面试主要涉及的几个方面。

1. 编程语言基础
	
	* 数据结构的实现细节以及比较：数组、链表、哈希表都是如何实现的？底层内存分配都是怎样的？插入与查找的时间复杂度是多少，分别有什么优缺点？
	* 编程语言特性：Java 的字符串常量池是怎么实现的？垃圾回收的流程又是怎么样的？
	* 关键字的特性：Java 中的 static 、final 关键字的含义以及使用场景。
	* 面向对象的细节：类的封装，函数与变量的继承，抽象类和接口有什么区别？
	* 并发编程：线程如何同步？进程如何同步？wait() 函数使用场景以及常用的并发编程模式。

2. 算法基础

	* 算法题：链表操作、二分查找、动态规划、DFS、BFS等等（可以利用 Leetcode 进行刷题训练）。
	* 算法复杂度的分析：时间复杂度、空间复杂度、平均时间复杂度。
	* 数据结构的实现：实现二叉查找树、Trie树等。

3. 计算机网络

	* 协议的基础组成和用途：HTTP 协议中不同头部、方法、状态码的含义。
	* 协议的使用场景：DNS、ARP相关协议的使用场景和原理。
	* 不同协议的区别：TCP 和 UDP 的区别，HTTP 和 HTTPS 的区别。
	* 协议具体功能的实现：TCP 三次握手原理，TCP 的四次挥手原理，以及各个状态之间的流转是怎样的？

4. 操作系统

	* 操作系统基础概念：进程，线程，虚拟内存，文件权限，信号量等概念考察。
	* 命令的考察：ls， find， top， ps 等命令的应用与原理。
	* 常见功能的实现：进程调度，用户态与内核态的切换，各类系统调用的实现方式，epoll 的实现。

5. 数据库：

	* 基本概念：事务，存储引擎，隔离级别，索引等概念考察。
	* 数据库基准测试与性能分析：基准测试的策略，性能剖析工具的使用和分析。
	* 数据库设计与优化：范式和反范式，索引的类型，如何选择合适的索引，如何使用硬件优化。
	* 数据库原理及高级特性：数据库分区分表，存储过程的使用，全文索引，主从热备等常用策略的使用与原理。

##### 项目经验

以上的技术基础部分相关的问题基本都是共通的，因为面试官很难跳出以上五大问题的框架。那么在项目经验这部分就是千人千面了。因为每个人的项目经历都不同，所以这部分其实就是靠你自己来主导将问题引导到你所熟悉的部分。所以在面试前，曾经参与过的项目需要认真回顾一遍，从技术选型，架构设计（即使是中途加入项目也应该对此有所了解），维护或者实现的功能细节，过程中遇到的技术难点，学到了什么知识，都可能被问到，常见的问题有：这个项目遇到了什么技术难点，你是如何解决的？这个项目你学习到了哪些技术知识点？

如果你有开源项目的话，其实也可以根据你的开源项目进行聊一些相关的技术，比如你是如何设计的？你这个开源项目运用了什么技术？解决了什么问题？等等。

##### 如何高效记忆这么多知识点？

以上我们简单讲述了准备阶段我们需要准备的知识，简单分为两类，一个是技术基础和项目经验，项目经验的话好说，因为都是自己的项目，那么如何说如何讲也没有一套固定的套路，只需要描述清楚即可。但是技术基础不一样，我们不但要用心理解，还要牢记于心。

背诵是必要的，对于面试的知识点来说，至关重要。很多知识我们不是特别理解，这时候也很难记下来，但是面试有可能会被问到，因此我们先背下来再说，也是为了方便我们日后的理解。

暂时抛开面试，单纯的为了学习知识，背诵也是有必要的。对于计算机来说，线程进程的概念、语言基础、数据库的事务、存储引擎、隔离级别、索引等等概念，先记住，背下来，在随后应用的时候再细细品味我们背过的知识就会有不同的理解。这些知识是信息密集度比较高的，也是经得起历史和时间的考验的，对于与初学者来说，我们很难一下子全部接收到其中丰富的内涵，因此需要先背下来，然后再慢慢反复思考和体会，不同的时期都会有不同层次的理解。

而且通过背诵来学习，是通过不断强化大脑对知识的感受，来使知识变得更加“自然”。通过理解来学习，是让新知识和已有的知识构成联系，进行一定的逻辑推导，来告诉自己这个知识是正确的，是“自然的”。所以通过背诵获得的知识是零散的，而通过理解获得的知识是整体的体系。也正是因为有了这些零散的知识才有助于我们获得整体的认识。因此无法理解并不是笨，而是忘记了一些已有知识，无法构成联系。

上面说了那么多，其实就是告诉大家对于知识点的记忆只有一个方法就是背下来。那么如何高效的背下来这么多的知识点呢？这里我给大家推荐一个软件 Anki，作为电子笔记本，它可以插入文字、图片（拍照/截图都可以）、音频等，基本上可以用来记录一切东西。Anki 非常利于复习记忆，它可以按照**艾宾浩斯遗忘曲线**，给你安排合理的复习频率，就像你使用背单词软件时的操作一样。

##### 模拟面试

如果你准备去面试一家你非常喜欢的公司，之前你也没有面试的经历，或者已经很久都没有经历过面试。那么你不能第一时间就直接投这家公司，因为如果你第一时间投了，有很大概率你会失败。因为有一些知识是你看再多的面试经验也学不到的。只有当实践真的如此去做，并且做到的时候才能称之为你的知识。所以一定要实践，可以找一些小公司去试试水，也可以找另一个工程师充当面试官。然后在面试过程中总结自己的不足，锻炼自己的面试技巧。当你面试多了以后就会发现其实所有的面试官问的问题都差不多，这时候你就可以向你心仪的岗位和公司发起冲锋了。

#### 交流阶段

一个正常的技术面试阶段，大致分为三个部分：

1. 自我介绍
2. 项目（技术）考察
3. 向面试官提问

##### 自我介绍

几乎所有的面试都是从自我介绍这个环节开始的，那么为什么自我介绍要作为一个面试的开场呢？

很普遍的问题就是面试官有可能是刚拿到你的简历，他需要在你自我介绍的时候快速浏览一下你的简历，因为技术面试的面试官很多都是一线的员工，面试候选人仅仅是工作的一小部分，所以很多情况下是没有提前看过你的简历。其次是自我介绍是个热身环节，面试官和候选人都是陌生人，自我介绍不管是面试还是其他的情况下，都是两个陌生人彼此交流的起点。最后自我介绍也是展示自我，引出接下来技术面试的引子，是你自己自己指定技术面试方向的一次机会。

自我介绍归根结底就是一个热身运动，所以切忌占用大量的时间。所以在面试前要提前准备一篇自我介绍的稿子，自我介绍时间最好是五分钟以内。如果时间过长的话会给面试官留下讲话缺乏重点、沟通能力一般的印象。

自我介绍中有以下的信息是必备的

* 个人信息：至少要体现出自己的姓名、工作年限和岗位。重点在于**你是谁**，如果是应届生并且学校比较好的话，也可以提及。如果是社招，有大厂经历的话最好也提及。这些都是加分项。
* 技术能力：简单的介绍自己的技术栈，熟悉哪方面的知识。重点在于**你会什么**，这里切忌把自己只是简单使用过的技术栈说出来。
* 技能擅长：重点介绍自己擅长的技术，比如性能优化、高并发、系统设计或者沟通协调的能力。重点在于**你擅长什么**。

在自我介绍中也要学会刻意引导，比如在你自我介绍中提及了一个项目中碰到了什么难点，然后被你攻克了，最后取得的效果是什么，如果你这么说一般面试官都会接着这个问题进行提问的。

##### 项目（技术）考察

一个好的技术考察的开始，必须得有自我介绍部分好的铺垫和引导。有一种情况我们经常遇见：候选人说了一大堆非重点的自我介绍，面试官一时语塞，完全 get 不到候选人的重点，也不知道候选人擅长什么、有什么亮点项目，然后就在他简历的技术栈中选了本公司也在用的技术，候选人这个时候也开始冒汗，因为这个技术栈并不是他的擅长，回答的也磕磕绊绊，面试官的引导和深入追问也没有达到很好的效果，面试就在这种尴尬的气氛中展开了，面试结束后面试官对候选人的评价是技术不熟练、没有深入理解原理，候选人的感受是，面试官专挑自己不会的问。

所以我们无论是在前面的自我介绍部分，还是简历部分都要有意的将面试官引向我们熟悉的技术方向以及擅长的领域。那么这里就要注意在准备我们擅长部分的答案的时候就不能准备的太浅显。比如 Java 中的 synchronized 关键字是面试中常常被问到的问题，那么这个问题如果你只是浅显得准备了如何使用，那么远远不够。你应该从如何使用到其字节码的表示以及跟随 Java 版本的升级引出的锁升级的四个状态到最后的操作系统原语的实现。这些你都要知道，而且还有其扩展知识，比如 synchronized 和 Lock 的区别、synchronized 锁升级引出的对象头的相关知识等等，这些你都要知道。

所以不能把自己准备的问题答案停留在一个很浅显的层面，一方面无法展示自己的技术深度，另一方面在面试官的提问中容易丢分，因此在自己的答案后面多进行自我追问，看能不能把问题做的更深入。

##### 向面试官提问

这个部分基本上到了技术面试的尾声，属于做好了不影响大局，但是做不好的话很容易踩雷的部分。这里向面试官提问有几个雷区需要注意。

1. 切忌问结果：问了也白问，如果你没过，面试官说了也很容易引起尴尬。
2. 切忌问工资：一般到了 HR 那一面会跟你谈工资，不要跟技术面试官聊工资相关的事。
3. 切忌问技术问题：不要拿自己不会的技术难题去为难面试官，没有意义。

有几个好的提问可以作为参考

1. 你对这个职位的理想人选要求是什么？
2. 所应聘的部门技术栈是什么？
3. 所应聘的部门技术人员有多少人？

尽量围绕你的岗位进行提问，这可以使得你更快得熟悉你的工作内容，也让面试官看到你对此岗位的兴趣和热情，重要的是这些问题对于面试官而言既可以简略回答，也可以详细的给你讲解，如果他很热情得跟你介绍此岗位相关的情况，说明你可能表现得不错，否则的话，你可能不在他的备选名单里，这个时候就需要你早做打算了。

### HR 面试

恭喜你，对于技术岗位 HR 面相对比较简单，只要你能够到达这一步，基本上表明了你已经通过了所有的面试。如果没有什么大问题的话在这一步是不会淘汰你的。所以为什么有的公司会连 HR 面都会省去，或者面试结束后先送走候选人，确认技术面通过以后隔天 HR 再电话沟通。

但是大公司这个过程是一定要有的，基本上就是为了确认你的沟通能力、诚实、稳定性等其他的软性技能。并确认你的薪资情况以及预期。接下来我就给大家说几个 HR 常问的问题我们该怎么答的例子。

#### 为什么离开上家公司？您在前一家公司的离职原因是什么？

该问题有设置陷阱，最好是选取既合理而又不得不辞职的理由，理由一：住处距离公司上班路程太远；理由二：个人原因，想换个环境。面试中这样的问题最不好回答，稍有不慎就有可能被 pass，比如：该公司内部提倡加班文化，而你的辞职理由中却说原公司加班太多。所以面试前需对公司有所了解，避免出现跟公司对冲的情况。

回答提示：

1. 最重要的是：应聘者要使招聘单位相信，应聘者在过往的单位的“离职原因”在此家招聘单位里不存在。
2. 避免把“离职原因”说得太详细、太具体。
3. 不能掺杂主观的负面感受，如“太辛苦”、“人际关系复杂”、“管理太混乱”、“公司不重视人才”、“公司排斥我们某某的员工”等。
4. 但也不能躲闪、回避，如“想换换环境”、“个人原因”等。
5. 不能涉及自己负面的人格特征，如不诚实、懒惰、缺乏责任感、不随和等。
6. 尽量使解释的理由为应聘者个人形象添彩。

相关例子：如“我离职是因为这家公司倒闭；我在公司工作了三年多，有较深的感情；从去年始，由于市场形势突变，公司的局面急转直下；到眼下这一步我觉得很遗憾，但还要面对显示，重新寻找能发挥我能力的舞台。”同一个面试问题并非只有一个答案，而同一个答案并不是在任何面试场合都有效，关键在应聘者掌握了规律后，对面试的具体情况进行把握，有意识地揣摩面试官提出问题的心理背景，然后投其所好。

分析：除非是薪资太低，或者是最初的工作，否则不要用薪资作为理由。“求发展”也被考官听得太多，离职理由要根据每个人的真实离职理由来设计，但是在回答时一定要表现得真诚。实在想不出来的时候，家在外地可以说是因为家中有事，须请假几个月，公司又不可能准假，所以辞职，这个答案一般面试官还能接受。

#### 你上家公司的薪水/期望的薪金？

解读：此处有陷阱，首先搞清他问你上家薪水的目的，据面试经验反馈，通常有两种情况，以之前的薪资作为压价的理由；做背调时考察你是否够诚实，有些大公司很喜欢做背调，通常hr都有内部群，“谁谁知道xx公司xx岗位的薪资水平是多少吗”，群里一问大概都清楚，所以不要夸的太离谱，适当水分可以。

正常情况来说，跳槽后涨薪是你上家公司薪水的百分之三十。所以如果你之前薪资是一万，你说你期望薪资是两万，那么基本上是很难争取到的。但是如果你之前薪资是一万五，你说你期望薪资是两万，那还是有的商量。那么此时或许有人会有疑惑，跳槽到新公司 HR 不是都会要求提供薪资证明吗？如果说高了你的薪资的话，你所说的和你的薪资证明对不上的话不会有问题吗？下面我就说一下该如何在合理的范围内对薪资增加水分。

比如你的基本薪资是8K，绩效薪资是基本薪资的一半4K，各种补助1K，五险一金中的公积金的比例是按照百分之六比例缴纳的。那么此时你会怎么回答这个问题呢？

* 回答上家公司薪水是8K：如果你这么说，那么正常情况下你的涨薪最多也就能达到11K。
* 回答上家公司薪水是13K：那么你的上限正常情况下就是17K。

这两个回答应该是最常见的回答，但是还有一种回答能够让你薪资在往上涨一截，那就是利用了上家公司公积金缴纳比例的问题，你可以说是按照百分之十二进行缴纳的，那么你上家公司薪水完全可以说成是14K，

还有一种情况是这样的，2019年的9月份你在原公司涨薪了，涨薪前基本薪资是8K，涨薪后基本薪资是10K，而你在2020年的四月份跳槽，那么此时你的薪资也可以加一点水分，因为每年因为工资调整而调整公积金缴纳基准是在七月份，所以四月份的薪资还是按照涨薪前的8K进行计算的。所以就可以根据你每月到手的钱进行反推你最高拿多少钱。

一个人因为懂得这些规则，相同的技术水平下，进入同一家公司，因为他说的上家薪水多，所以会比你多个两三千都有可能。因此你应该懂得这些计算规则，不然吃亏的还是你。

#### 自己的优点和缺点是什么？并举例说明？

优点回答提示：沉着冷静、条理清楚、立场坚定、顽强向上、乐于助人和关心他人、适应能力和幽默感、乐观和友爱。

缺点回答提示：这个问题企业问的概率很大，通常不希望听到直接回答的缺点是什么等，如果求职者说自己小心眼、爱忌妒人、非常懒、脾气大、工作效率低，企业肯定不会录用你。绝对不要自作聪明地回答“我最大的缺点是过于追求完美”，有的人以为这样回答会显得自己比较出色，但事实上，他已经岌岌可危了。企业喜欢求职者从自己的优点说起，中间加一些小缺点，最后再把问题转回到优点上，突出优点的部分，企业喜欢聪明的求职者。

#### 你为什么愿意到我们公司来工作？

回答提示：对于这个问题，你要格外小心，如果你已经对该单位作了研究，你可以回答一些详细的原因，像“公司本身的高技术开发环境很吸引我。”、“我同公司出生在同样的时代，我希望能够进入一家与我共同成长的公司。”、“你们公司一直都稳定发展，在近几年来在市场上很有竞争力。”、“我认为贵公司能够给我提供一个与众不同的发展道路。”这都显示出你已经做了一些调查，也说明你对自己的未来有了较为具体的远景规划。

### 总结

所有的奇淫技巧都是锦上添花，如果你本人只有半吊子水平的话，那么无论再怎么包装自己终有被识破的那一天。所以在平常的时候要多看书、多总结、多思考、多实践。

最后预祝大家都能够找到自己心仪的岗位和公司。
